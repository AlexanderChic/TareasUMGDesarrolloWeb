📖 Descripción del Curso

El curso de Bases de Datos proporciona una comprensión integral de los sistemas de gestión de bases de datos relacionales y no relacionales. Los estudiantes aprenden a diseñar, implementar y administrar bases de datos eficientes, así como a utilizar lenguajes de consulta como SQL para manipular y extraer información. El curso cubre desde conceptos teóricos fundamentales hasta aplicaciones prácticas en el desarrollo de sistemas de información.



🎯 Puntos Clave del Aprendizaje

🏗️ Modelado de Datos

Diseño conceptual, lógico y físico de bases de datos usando diagramas ER y técnicas de normalización.

⚡ SQL Avanzado

Dominio completo de consultas SQL, procedimientos almacenados, triggers y optimización de consultas.

🔒 Integridad de Datos

Implementación de restricciones, claves primarias, foráneas y reglas de negocio para garantizar consistencia.

👨‍💼 Administración de SGBD

Gestión de usuarios, permisos, respaldos, recuperación y mantenimiento de sistemas de bases de datos.

🔄 Transacciones y Concurrencia

Control de transacciones, propiedades ACID, bloqueos y manejo de acceso concurrente a datos.

🌟 Bases de Datos NoSQL

Introducción a bases de datos documentales, clave-valor y orientadas a grafos para aplicaciones modernas.

📐 Formas Normales

1️⃣ Primera Forma Normal (1FN)

Elimina grupos repetitivos y asegura que cada atributo contenga valores atómicos indivisibles.

2️⃣ Segunda Forma Normal (2FN)

Cumple 1FN y elimina dependencias funcionales parciales respecto a la clave primaria.

3️⃣ Tercera Forma Normal (3FN)

Cumple 2FN y elimina dependencias transitivas entre atributos no clave.

🏆 Forma Normal Boyce-Codd (FNBC)

Versión más estricta de 3FN que maneja casos especiales de dependencias funcionales.


💾 Ejemplo de Consulta SQL Compleja

-- Consulta para obtener el top 5 de productos más vendidos por categoría
-- con información de ventas y clientes

SELECT 
    c.nombre_categoria,
    p.nombre_producto,
    SUM(dv.cantidad) as total_vendido,
    SUM(dv.cantidad * dv.precio_unitario) as ingresos_totales,
    COUNT(DISTINCT v.id_cliente) as clientes_unicos,
    AVG(dv.precio_unitario) as precio_promedio
FROM 
    productos p
    INNER JOIN categorias c ON p.id_categoria = c.id_categoria
    INNER JOIN detalle_ventas dv ON p.id_producto = dv.id_producto
    INNER JOIN ventas v ON dv.id_venta = v.id_venta
WHERE 
    v.fecha_venta >= DATE_SUB(CURDATE(), INTERVAL 1 YEAR)
GROUP BY 
    c.id_categoria, c.nombre_categoria, p.id_producto, p.nombre_producto
HAVING 
    total_vendido > 100
ORDER BY 
    c.nombre_categoria, total_vendido DESC
LIMIT 5;
